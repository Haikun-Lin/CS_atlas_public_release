# Execution manifest

This manifest summarizes how the public repository connects downstream
analytical notebooks, private processed-object requirements, standalone CellChat
scripts, and release-facing documentation.

The repository is downstream and object-centered. It does not distribute raw
sequencing data, raw count matrices, patient-level private information, or the
processed AnnData objects used by the downstream workflows. Public quantitative
results, QC/cell-count summaries, gene-set/module definitions, and selected
expression summaries are provided through the manuscript supplementary tables
and figures. The object schema reports in `docs/processed_object_schemas/`
document the structure of the private processed objects without exposing
expression matrices or identifiers.

## Environment and path-variable documentation

The recommended compact environment record is:

- `environment/methods_software_table.csv`
- `environment/python_requirements_public.txt`

The scripts used to regenerate environment records on the analysis server are:

- `environment/export_python_environment.py`
- `environment/export_cellchat_r_environment.R`

See `environment/README.md` for public-release guidance.

Private input and generated-output locations are documented through the
environment-variable names recorded in notebook setup cells, CellChat scripts,
and the workflow sections below.

## Notebook 01: global atlas

File:

- `notebooks/01_global_atlas_analysis.ipynb`

Purpose:

- Load and validate the integrated immune-cell atlas object.
- Inspect object metadata, QC fields, sample-type mapping, and annotation
  structure.
- Generate global atlas composition, UMAP, Sankey, cell-count, QC, and
  integration-quality outputs.
- Compute cytokine-storm gene expression summaries, enrichment outputs, module
  scores, correlations, and global visualization panels.
- Prepare and validate CellChat input folders for CAR-T, COVID-19, and SLE.
- Perform selected SLE pDC/IFN analyses retained in the global-atlas workflow.

Private inputs:

- Processed integrated AnnData object configured by `ATLAS_H5AD`.
- Sample metadata workbook configured by `SAMPLE_METADATA_XLSX`.

Documented path variables:

- `ATLAS_OUTPUT_DIR` controls global-atlas outputs.
- `CELLCHAT_ROOT`, `CELLCHAT_CAR_T_INPUT_DIR`, `CELLCHAT_COVID19_INPUT_DIR`,
  and `CELLCHAT_SLE_INPUT_DIR` control CellChat input exports.

Major public/release-facing records:

- `docs/processed_object_schemas/global_atlas_schema.json`
- manuscript supplementary tables/figures for QC, cell counts, gene sets,
  expression summaries, and selected statistical outputs.
- CellChat input-export summaries generated by the notebook.

## Notebook 02: myeloid and monocyte analysis

File:

- `notebooks/02_myeloid_monocyte_analysis.ipynb`

Purpose:

- Load and validate the processed myeloid/monocyte object.
- Inspect myeloid annotations and UMAP structure.
- Analyze monocyte subcluster composition across disease, stage, and severity
  groups.
- Compute cytokine-storm scores and functional-module summaries.
- Run preserved DEG and enrichment workflows for cross-disease, COVID-19, SLE,
  CD16-monocyte, and CAR-T-associated monocyte analyses.
- Generate disease-specific monocyte functional-state, IFN, receptor, and
  correlation outputs.

Private input:

- Processed myeloid/monocyte AnnData object configured by `MYELOID_H5AD`.

Documented path variables:

- `MYELOID_OUTPUT_DIR` controls myeloid-analysis outputs.

Major public/release-facing records:

- `docs/processed_object_schemas/myeloid_monocyte_schema.json`
- manuscript supplementary tables/figures covering relevant QC/cell-count,
  gene-set/module, expression, DEG, enrichment, and plotted summary results.

Private/intermediate outputs not distributed by default:

- Processed myeloid/monocyte AnnData object.
- Cell-level expression matrices and identifiers.
- Temporary reconstructed objects used inside preserved analysis sections.
- Notebook-generated intermediate files not selected for manuscript or release.

Notes:

- The notebook intentionally preserves cell-level DEG designs specified by the
  analysis plan.
- The object requires a populated `.raw` gene space for preserved DEG/scoring
  sections.

## Notebook 03: T-lineage and CAR-T analysis

File:

- `notebooks/03_t_lineage_cart_analysis.ipynb`

Purpose:

- Load and validate the processed T-lineage object.
- Identify CAR-T and endogenous T-cell (`endoT`) populations using the approved
  sample mapping.
- Generate T-cell subcluster and CAR-T visualization outputs.
- Compute T-lineage and CAR-T cytokine-storm scores and expression summaries.
- Run CAR-T DEG and enrichment comparisons.
- Analyze CAR-T versus `endoT` subcluster dynamics, functional states, radial
  plots, CD4/CD8 CAR-T transitions, and stage-specific summaries.

Private input:

- Processed T-lineage AnnData object configured by `T_LINEAGE_H5AD`.

Documented path variables:

- `T_LINEAGE_OUTPUT_DIR` controls T-lineage-analysis outputs.

Major public/release-facing records:

- `docs/processed_object_schemas/t_lineage_car_t_schema.json`
- manuscript supplementary tables/figures covering relevant cell-count,
  gene-set/module, expression, DEG, enrichment, and plotted summary results.

Private/intermediate outputs not distributed by default:

- Processed T-lineage AnnData object.
- Cell-level expression matrices and identifiers.
- Notebook-generated intermediate files not selected for manuscript or release.

Notes:

- `CAR-T_anno` is constructed and validated in the notebook from the approved
  sample mapping.
- `CAR-T` denotes infused CAR-T cells; `endoT` denotes endogenous T cells.

## CellChat input preparation

CellChat input folders are exported by `notebooks/01_global_atlas_analysis.ipynb`.
Each group-specific input folder contains:

- `expression_matrix.mtx`
- `genes.csv`
- `meta.csv`

The required metadata column is:

- `cellchat_anno`

Default input roots:

- `outputs/cellchat/cart/input/`
- `outputs/cellchat/covid19/input/`
- `outputs/cellchat/sle/input/`

The notebook validates nonempty groups, gene/cell dimensions, annotation
availability, and exported matrix shape before writing CellChat inputs.

## Release and privacy notes

- Processed AnnData objects are private execution inputs and are not included.
- Raw sequencing data and raw count matrices are outside the scope of this
  downstream code repository.
- Private sample identifiers, patient identifiers, and cell barcodes should not
  be committed.
- The manuscript supplementary tables provide release-facing QC, cell-count,
  gene-set/module, expression, and selected statistical summaries.
- Intermediate notebook outputs are not all distributed separately; they are
  regenerated from private processed objects when the notebooks are run.
- Any local/server paths should be provided through environment variables or
  local ignored configuration files, not committed into the public repository.
